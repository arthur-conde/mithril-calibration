# gorgon-calibration

Community-sourced calibration data for [Project Gorgon companion app](https://github.com/arthur-conde/project-gorgon)'s **Samwise** (garden growth rates) and **Arwen** (NPC favor/gift rates) modules. New installs of the app fetch the aggregated rates here so they get useful forecasts on day one instead of grinding from zero.

## What's in this repo

| Path | Purpose |
|---|---|
| `aggregated/samwise.json` | Current merged Samwise rates, consumed by the app |
| `aggregated/arwen.json` | Current merged Arwen rates, consumed by the app |
| `contributions/samwise/` | Raw per-contributor Samwise submissions (one `.json` per contribution) |
| `contributions/arwen/` | Raw per-contributor Arwen submissions |
| `scripts/aggregate.py` | Merges `contributions/*` into `aggregated/*` |
| `.github/ISSUE_TEMPLATE/` | Submission templates wired into the app's `Share` button |
| `.github/workflows/aggregate.yml` | CI runs the aggregator on new issues and PRs |

The app fetches from `https://raw.githubusercontent.com/arthur-conde/gorgon-calibration/main/aggregated/{samwise|arwen}.json`.

## Contributing

The app's calibration tab has a `Share with community` button. It:
1. Exports your local rates as a sanitized JSON payload (rates, sample counts, min/max — **no** raw observations, **no** character names).
2. Copies the JSON to your clipboard and opens a pre-filled GitHub issue.
3. Paste the JSON into the issue body and submit.

The aggregator picks up new issues, merges your numbers with everyone else's using sample-count-weighted averages, and regenerates `aggregated/*.json`. Your data never leaves this repo.

### Payload shape

**Samwise** (`schemaVersion: 1`):

```jsonc
{
  "schemaVersion": 1,
  "module": "samwise",
  "exportedAt": "2026-04-22T...",
  "contributorNote": "optional",
  "rates":        { "Carrot": { "avgSeconds": 3612, "sampleCount": 12, "minSeconds": 3480, "maxSeconds": 3720 } },
  "phaseRates":   { "Carrot|Planted→Growing": { "avgSeconds": 1800, "sampleCount": 8, "minSeconds": 1740, "maxSeconds": 1860 } },
  "slotCapRates": { "Lily": { "observedMax": 8, "sampleCount": 3 } }
}
```

**Arwen** (`schemaVersion: 2`):

```jsonc
{
  "schemaVersion": 2,
  "module": "arwen",
  "exportedAt": "2026-04-22T...",
  "contributorNote": "optional",
  "itemRates":      { "NPC_Makara|StafflordsRing":            { "rate": 0.1625, "sampleCount": 1, "minRate": 0.1625, "maxRate": 0.1625 } },
  "signatureRates": { "NPC_Makara|Rare or Better Magic Gear": { "rate": 0.1070, "sampleCount": 29, "minRate": 0.0006, "maxRate": 0.4109 } },
  "npcRates":       { "NPC_Makara":                           { "rate": 0.1070, "sampleCount": 29, "minRate": 0.0006, "maxRate": 0.4109 } },
  "keywordRates":   { "MinRarity:Rare":                       { "rate": 0.1070, "sampleCount": 29, "minRate": 0.0006, "maxRate": 0.4109 } }
}
```

## Aggregation math

- `avg` / `rate`: sample-count-weighted mean across contributors.
- `sampleCount`: sum across contributors.
- `min` / `max`: global min / max across contributors.
- `SlotCapRate.observedMax`: global **max** (caps are ceilings, not averages).

Aggregation runs per-key within each dictionary. Dictionaries never merge across types.
